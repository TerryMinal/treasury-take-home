# Contributing

## Before Opening a Pull Request

Make sure to:

-   Format the code.
-   Run the linter.
-   Run the type checker.
-   Run the test suite.
-   Update documentation if behavior has changed.

## Code Style

-   Keep pull requests focused on a single change.
-   Use descriptive names for variables, functions, and files.
-   Remove unused imports and dead code.
-   Do not commit secrets, credentials, or API keys.
-   Avoid commented-out code.

## Testing

-   Every behavior change should include appropriate tests.
-   Bug fixes should include regression tests whenever practical.
-   Prefer deterministic tests that do not rely on external services.

## Documentation

Update documentation whenever you:

-   Change APIs
-   Change architecture
-   Introduce new dependencies
-   Add significant features

## Review Checklist

Before requesting review, verify that:

-   Code is formatted
-   Linting passes
-   Type checking passes
-   Tests pass
-   Documentation is up to date
-   No unrelated changes are included
